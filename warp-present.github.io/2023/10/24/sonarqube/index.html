Sonarqube

如何在本地端使用Sonarqube

  1. 下載到local

  2. docker
    https://hub.docker.com/_/sonarqube

    下載映像檔：
    docker pull sonarqube

    啟動docker
    docker run -d –name sonarqube -e SONAR_ES_BOOTSTRAP_CHECKS_DISABLE=true -p 9000:9000 sonarqube:lates

  3. 開啟localhost:9000

  4. 登入(預設帳號密碼 adimin/admin)

  5. 選擇建立新專案，設定專案名稱

  6. 選擇local(非免費版有其他功能)

  7. 取得token，跟著步驟進行

在local端使用Sonarqube

  1. 下載sonar scaner 並設定path

    下載程式
    https://docs.sonarsource.com/sonarqube/latest/analyzing-source-code/scanners/sonarscanner/

    設定path
    path+=(‘{檔案位置}/bin’) export PATH

    sonar-scanner
    -Dsonar.projectKey={projectKey}
    -Dsonar.sources=.
    -Dsonar.host.url=http://localhost:9000
    -Dsonar.token={token}

  2. 使用docker scanner-cli
    下載image
    指令：
    docker run
    –rm
    –network=host
    -e SONAR_HOST_URL=’http://localhost:9000‘
    -e SONAR_SCANNER_OPTS=”-Dsonar.projectKey={projectKey}”
    -e SONAR_LOGIN=”token”
    -v “{專案位置}:/usr/src”
    sonarsource/sonar-scanner-cli

專案設定

專案內加入sonar-project.properties
根據需求設定
ex:
sonar.projectKey=jest-github
sonar.sources=src/
sonar.test.inclusions=**/*.test.tsx
sonar.testExecutionReportPaths=sonar-report.xml
sonar.javascript.lcov.reportPaths=./coverage/lcov.info

透過repo ex:github action

  1. 取得token
  2. 設定yaml檔案(在上repo前先使用scanner掃描)
  3. 如果是local，務必要把port接出去(ex:ngrok)
  4. 上傳到github repo

加上測試報告

根據要整合的測試框架下載plugin

jest:
jest-sonar-reporter

vitest:
vitest-sonar-reporter

目的:產生合規格的xml檔案，以及lcov.info檔案

根據plugin的說明安裝設定
在sonar-project.properties裡加上設定
ex:
sonar.test.inclusions=**/*.test.tsx(選擇測試檔案名稱)
sonar.testExecutionReportPaths=sonar-report.xml(測試結果xml檔名稱)
sonar.javascript.lcov.reportPaths=./coverage/lcov.info(讓scanner讀取測試檔)

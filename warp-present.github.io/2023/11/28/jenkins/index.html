什麼是Jenkins:

Jenkins是CI&CD開源工具， 可以設定編譯、佈署的參數，並觸發整個 CI/CD 流程。除了透過手動觸發外，也可以使用 Cron-Style 排程、或是串接github,gitlab等

本地端使用：

  1. 安裝
    mac: brew install jenkins
    docker: https://hub.docker.com/r/jenkins/jenkins
  2. 開啟(預設port:8080)
    mac: brew services start jenkins
  3. 開啟網頁
    http://localhost:8080/

pipeLine

這個方式可以把所有設定都寫在專案裡jenkinsfile，讓步驟變成一個一個的stage，並且可以串接github webhook，讓jenkins可以自動觸發

pipeline 基本語法

  1. agent
    指定整個 pipeline 或特定的 stage 將執行在 jenkins 的哪個位置

     agent 的參數
 a. any
 在任何可用的機器上執行 pipeline

 b. none
 設置為 none 的話代表不設定 global 的 agent, 這樣的話就要在 stage 上去設定 agent

  2. stage
    pipeline 的執行階段, 由多個 steps 所組成

  3. steps
    pipeline 主要執行的任務步驟

  4. environment
    設定變數給 stage 使用

  5. when
    可以用來執行程式碼執行的邏輯

  6. tools
    設定工具給 stage 使用

與git整合(以github為例)

  1. 如果jenkins在local，務必要把port接出去(ex:ngrok)

  2. 設定jenkins作業

  3. github webhook設定

     在settings -> webhooks -> add webhook

 https://{帳號}:{帳號token}@{server url}/job/jenkins_pipe/build?token={自訂token}

 (帳號token: profile -> developer settings -> personal access tokens -> generate new token)

  4. push上github，jenkins會自動觸發
